## 1. 背景 & 定位

v1.0 Dashboard 功能上线 + Temu 审核提交,但 UI 过于朴素,无法传达"跨平台 / 多店铺 / 多维度 BI"的产品愿景。用户提供了一版 AI 模型设计稿(电商运营 BI 数据看板),作为 v1.1+ 的视觉目标。

### 为什么要分 10 个 sub

模型设计稿整体做完是 10 个子系统级别的工作(Dashboard 首页 + 9 个 sidebar 分析页),一次实现风险高、周期长、且多数页面缺真实数据源(广告/订单/成本/漏斗)。策略:

- **Sub A + Sub B** 第一步做(v1.1):重做 MainLayout + 首页,完整视觉对齐设计稿,所有真数据都填,缺数据的区域做优雅占位
- Sub C–J 作为 sidebar 路由入口 + `<ComingSoon />` 占位页,保留视觉完整性,待后续逐个升级

### 一句话

**v1.1 是"视觉升级版的 v1.0"** — 布局换到设计稿样式,功能保持 v1.0 现有(件数 BI + 库存预警 + 核价提醒),所有缺数据点用统一灰色占位 + tooltip 说明目标版本。

## 3. 关键设计决策

| # | 决策 | 选中 | 理由 |
|---|---|---|---|
| 1 | 占位样式 | **灰色 `—` + tooltip 说明 + 整卡淡灰 + 虚化 sparkline** | 符合专业 BI(Grafana / Tableau 空值都是 `—`);hover tooltip 解释具体缺什么 |
| 2 | 时间范围下拉 | **5 项全显,今/7/30 可用,本月/自定义 disabled + tooltip「v1.2 订单 API」** | 跟决策 1 占位语言一致;UI 贴图 |
| 3 | Sidebar 配置项去留 | **9 项分析 + 头像下拉(店铺管理 / 设置 / 退出)** | 符合 BI 工业标准;sidebar 专注分析;router guard 兜底新用户 onboarding |
| 4 | 多平台/多区域筛选 | **chip 全显,非 Temu / CN/PA 之外灰 disabled + tooltip** | 视觉贴图;多平台 vision 明确传达 |
| 5 | 搜索框 | **前端 fuzzy(shops + topSkus)** | 零后端改动;覆盖"快速跳店"常见场景;大规模搜索等 V1.5 |
| 6 | 商品缩略图 | **不做,文字 SKU 名代替** | 额外 Temu API 开销太大;BI 专业产品不依赖缩略图 |
| 7 | KPI CTR vs 低库存 | **保留图 8 KPI 完整性,CTR 占位;低库存放 Zone 9 异常预警** | 图完整性 > 单一卡片可见性;异常预警区位置显眼 |
| 8 | 筛选触发 refetch | **debounce 300ms + 前序请求 abort** | 避免连续点筛选产生大量请求 |
| 9 | Sidebar 折叠记忆 | **localStorage 跨会话持久** | 用户习惯保留 |

## 6. 后端 API 改动

### `GET /api/dashboard/summary` 扩展

**新 query 参数**:
```
platform?:    'temu' | 'tiktok' | 'mercadolibre' | 'shopee' | 'amazon'  (现只接 temu)
region?:      'cn' | 'pa'  (现只接这两个)
shopIds?:     逗号分隔 UUID 数组
timeRange?:   'today' | '7d' | '30d'  (决定 orderBy 字段)
```

**响应扩展**:
```typescript
{
  kpis: {                                // 对应 Zone 1 的 8 张 KPI 卡
    salesVolume: number,                 // 按 timeRange 取 today/7d/30d 对应字段,求和
    // 占位字段统一返 null,前端渲染灰色 `—`
    gmvCents: null,                      // 2 GMV
    orderCount: null,                    // 3 订单量
    netProfitCents: null,                // 4 净收益
    grossProfitCents: null,              // 5 毛利润
    grossMarginPct: null,                // 6 毛利率
    adSpendCents: null,                  // 7 广告花费
    roas: null,                          // 8 ROAS
  },
  salesTrend: {                          // 3 点:今/7/30
    today:   { volume: number, gmvCents: null, orderCount: null },
    last7d:  { volume: number, gmvCents: null, orderCount: null },
    last30d: { volume: number, gmvCents: null, orderCount: null },
  },
  platformComparison: [                  // 只有 Temu 一行
    { platform: 'temu', salesVolume: number, gmvCents: null, orderCount: null },
  ],
  shopRanking: Array<{                   // 按销量降序
    shopId, shopName, platform, salesVolume, gmvCents: null, changePct: null,
  }>,
  regionDistribution: [                  // 按 shop.region 分组
    { region: 'cn', salesVolume: number, shopCount: number },
    { region: 'pa', salesVolume: number, shopCount: number },
  ],
  topSkus: Array<DashboardSkuRow>,       // 按 timeRange 对应销量排前 10
  productDetails: {                      // Zone 8 分页
    total: number,
    page: number,
    pageSize: number,
    items: Array<DashboardSkuRow>,
  },
  alerts: {
    lowStockCount: number,               // 真
    lowRoiCount: null,                   // 占位
    shopDeclineCount: null,              // 占位
  },
  pendingPriceReviews: number,
  dataFreshness: string | null,
}
```

### `DashboardService.summary()` 修改

- 新入参 `filters: { platform?, region?, shopIds?, timeRange? }`
- where 子句加 `region` / `shopId in shopIds` 过滤
- orderBy TopSku 按 timeRange 切换字段(`todaySaleVolume` / `sales7dVolume` / `sales30dVolume`)
- 新 aggregate:`regionDistribution`(groupBy region + sum)、`shopRanking`(groupBy shopId + sum)、`platformComparison`(目前只有 temu,一行)
- 保持 `orgId` 强隔离

## 8. 前端交互规则

### FiltersStore (Pinia)

```typescript
export const useFiltersStore = defineStore('filters', () => {
  const platform = ref<string>('temu');            // 默认 Temu
  const shopIds = ref<string[]>([]);                // 多选,空 = 全部
  const region = ref<'cn' | 'pa' | null>(null);    // null = 全部
  const timeRange = ref<'today' | '7d' | '30d'>('30d');   // 默认近 30 日

  // 任一改动触发 dashboard refetch(debounce 300ms)
  watch([platform, shopIds, region, timeRange], debounce(() => {
    useDashboardStore().fetch();
  }, 300));

  return { platform, shopIds, region, timeRange };
});
```

### Sidebar 折叠

```typescript
// MainLayout.vue
const collapsed = ref(localStorage.getItem('sidebar-collapsed') === 'true');
watch(collapsed, (v) => localStorage.setItem('sidebar-collapsed', String(v)));
```

### 全局搜索(前端 fuzzy)

```typescript
// GlobalSearch.vue
const q = ref('');
const results = computed(() => {
  const needle = q.value.toLowerCase();
  if (!needle) return [];
  const shops = useShopsStore().items
    .filter(s => s.displayName?.toLowerCase().includes(needle) || s.platformShopId.includes(needle))
    .slice(0, 10)
    .map(s => ({ type: 'shop', ...s }));
  const skus = useDashboardStore().data?.topSkus ?? [];
  const skuMatches = skus
    .filter(s => s.skuTitle?.toLowerCase().includes(needle) || s.platformSkuId.includes(needle))
    .slice(0, 10)
    .map(s => ({ type: 'sku', ...s }));
  return [...shops, ...skuMatches];
});
```

点击结果:
- shop → `router.push('/shops/' + id)`(现 /shops 列表页,暂无 detail 页;v1.2 扩展)
- sku → 关闭搜索下拉 + 滚动到商品明细表并高亮对应行

## 9. 范围边界 & 风险

### 做 ✅(§ 2 目标全部)

### 不做(§ 2 非目标全部)

### 已知风险

1. **信息密度高**:10 zone 一屏,响应式要精细。**建议最小桌面 1440px**,低分辨率堆叠。
2. **筛选 refetch**:debounce 300ms + 前序请求 abort(使用 `AbortController`)。
3. **占位 sparkline 违和**:预置 2-3 条不同 path 随机分配降低整齐感。
4. **低库存从 KPI 移到 Zone 9**:sidebar「库存预警」菜单加 badge 数字弥补。
5. **7 占位路由体验像 demo**:每个占位页标题 + 图标 + 说明不同;Footer 给 feedback 按钮(mailto)收集优先级意见。
6. **后端响应结构变化**:旧 `/summary` 返回字段(`kpis.todayVolume` 等)不保留,前端旧 Dashboard 如果恢复引用会断;但旧 Dashboard 本次会被重写,不影响。
7. **审核版本 vs v1.1**:Temu 审核目前用 v1.0 快照(public-beta 部署的版本)。v1.1 改动**先不 push 到公测域名**,本地开发完整验证后再决定是否替换审核版本。
