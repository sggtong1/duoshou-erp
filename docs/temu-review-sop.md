# 舵手 ERP — Temu 应用市场审核 SOP

- **应用名称**: 舵手ERP
- **应用定位**: 跨店聚合 BI 看板(销售 + 库存 + 核价提醒)
- **测试地址**: https://duoshouerp.868818.xyz
- **应用类型**: 只读数据分析工具,不执行任何写入类操作
- **目标用户**: 管理多家 Temu **全托管**店铺的中小商家 / MCN / 品牌代运营团队
- **版本**: v1.0(MVP)

---

## 一、产品价值

Temu 官方卖家中心是「单店视角」—— 同一个商家如果管理 5 家店铺,要登录/切换 5 次才能看完销量、库存、待办事项。舵手 ERP 解决这个痛点:

**一次登录,看到全部店铺的聚合数据**

- 今日销量汇总(所有店合计)
- 近 7 日 / 近 30 日销量对比
- 低库存告警(按 SKU 剩余销售天数排序,优先补货)
- TOP 20 热销 SKU(跨店销量排行,识别爆品)
- 跨店核价单待办提醒

商家每天花 30 秒扫一眼 Dashboard 即可掌握整体运营态势,不用再各店逐一登录。

## 二、审核测试步骤

### Step 1:登录

- 访问测试地址:https://duoshouerp.868818.xyz
- 使用提供的测试账号 + 密码登录
- 系统会自动跳转到店铺管理页(首次登录,强制 onboarding)

### Step 2:店铺连接(onboarding)

如果测试账号已预连接店铺,**可跳过此步,直接到 Step 3**。若是全新账号:

1. 点「➕ 添加店铺」
2. 填写 Temu 全托管店铺凭据:
   - 店铺类型:**全托管**
   - 区域:**美国站(PA)**
   - Platform Shop ID(Temu 卖家中心里看到的店铺 ID)
   - App Key / App Secret / Access Token(从 Temu 服务商应用管理后台生成)
   - 店铺显示名(自定义,便于识别)
3. 点「🔍 测试连接」— 后端调 `bg.mall.info.get` 验证凭据,返回店铺信息
4. 测试通过后点「💾 保存」
5. 系统自动后台触发首次数据同步(调 `bg.goods.salesv2.get` 拉 SKU 级销量 + 库存快照)

### Step 3:Dashboard — 核心功能页

左侧导航点「📊 Dashboard」(或直接访问 `/`):

**顶部**:
- 店铺筛选下拉(默认「全部店铺」合计,可切换单店)
- 数据最后同步时间戳
- 「🔄 立即同步」按钮(触发即时拉取)

**4 个 KPI 卡片**:
- 今日销量(件)
- 近 7 日销量(件)
- 近 30 日销量(件)
- 低库存 SKU 数(按「设置」页的阈值计算,默认 10 件)

**2 张 ECharts 图表**:
- **销售对比柱状图**:今日 / 7 日 / 30 日三段对比
- **TOP 20 近 30 日销量水平条图**:按 SKU 销量降序

**2 张数据表**:
- **TOP 10 今日热销**(SKU / 店铺 / 今日销量 / 库存)
- **库存预警**(SKU / 店铺 / 当前库存 / 日均销量 / 剩余天数,按剩余天数升序;<3 天红标,<7 天黄标)

**右下角 floating 提示**:如有待处理核价单,显示数量并可点击跳转。

### Step 4:店铺管理

左侧点「🏪 店铺管理」:
- 列表展示所有已连店铺(显示名 / Platform Shop ID / 类型 / 区域 / 状态)
- 可对 active 店铺点「断开」→ 软删(状态改 `disconnected`,历史数据保留,同步任务停止)

### Step 5:我的商品(只读列表)

左侧点「📦 我的商品」:
- 跨店商品列表,支持按店铺筛选
- 数据来自 `bg.glo.goods.list.get` / `bg.goods.list.get`(按 region)
- **本页仅展示,不提供发布/编辑/批量操作**

### Step 6:核价提醒(只读)

左侧点「🔔 核价提醒」:
- 页面顶部 banner 明确说明:「当前模块为只读提醒视图。批量同意/拒绝请到 Temu 卖家中心操作」
- 展示所有待处理核价单列表(合并所有店铺)
- 每行可点「详情」进入单个核价单只读视图
- **不提供任何写入操作**(不调 `.confirm` / `.reject` 接口)

### Step 7:设置

左侧点「⚙️ 设置」:
- 修改「低库存阈值」(默认 10 件,0-100000 可调)
- 修改「剩余天数告警阈值」(默认 7 天)
- 保存后,Dashboard 的低库存 SKU 数和表格颜色会立即按新阈值刷新

### Step 8:数据同步

- **自动同步**:后端每小时自动调 `bg.goods.salesv2.get` 拉取最新 SKU 销售+库存快照
- **手动触发**:Dashboard 右上角「立即同步」按钮,返回 202 Accepted,后台 fire-and-forget 执行
- **前端轮询**:触发同步后每 5 秒刷新一次 summary,最多 12 次(1 分钟),感知数据到位即停

## 三、技术边界(审核参考)

### 本应用**会**调用的 Temu API

| API | 用途 | 频率 |
|---|---|---|
| `bg.mall.info.get` | 店铺连接时验证凭据 + 检查 shopType 一致 | 每次用户连店一次 |
| `bg.goods.salesv2.get` | SKU 销量 + 库存快照同步(Dashboard 数据源) | 每店每小时 1 次,分页至 total |
| `bg.glo.goods.list.get` / `bg.goods.list.get` | 商品列表同步(「我的商品」页) | 每店每 30 分钟 1 次 |
| `bg.price.review.page.query` / `bg.semi.price.review.page.query.order` | 核价单列表查询(只读展示) | 每天 1 次 |

### 本应用**不会**调用的 Temu API(本 MVP 明确不做)

- **写入类**:`bg.goods.add` / `bg.goods.edit` / 各种发布/编辑/删除接口
- **价格调整**:`bg.semi.adjust.price.batch.review` / `bg.full.adjust.price.batch.review`
- **核价操作**:`.confirm` / `.reject` 系列
- **活动报名**:`bg.marketing.activity.enroll.submit`

### 多店并发

- 每店调用通过 Redis 限流(qps: 5, burst: 5)
- Cron 使用 SETNX 全局锁,避免同一任务多进程并发

### 数据存储

- PostgreSQL(Supabase 托管)
- Temu 凭据(App Key / Secret / Access Token)**AES-256-GCM 加密**后存储,`CREDS_ENCRYPTION_KEY` 仅在服务端环境变量内
- 前端 bundle 不包含任何 Temu 凭据或服务端密钥
- 多租户严格隔离:每条数据库查询都带 `orgId` 过滤

## 四、安全与合规

### 认证

- 用户侧:Supabase Auth(邮箱/密码,支持邮件验证)
- API 侧:所有非 `/login` 路径由 `AuthGuard` 保护,Bearer JWT Token

### 传输

- 全站 HTTPS(Let's Encrypt 免费证书,每 90 天自动续期)
- API 反向代理通过 nginx,后端 API 不直接暴露端口

### 隐私政策

- 不向第三方转发或出售任何 Temu 店铺数据
- 数据仅用于:(a) 向授权商家自身展示聚合分析结果;(b) 服务端内部运营日志和错误监控
- 商家可随时在「店铺管理」里断开店铺,停止后续同步

### 凭据管理

- 创建店铺时凭据立即加密入库,原文 access token 不落日志
- 「断开」操作不物理删除加密凭据(软删以保留重连能力),但 cron 会因 `status=disconnected` 跳过该店

## 五、常见问题

**Q: 审核账号登录后 Dashboard 是空的?**

A: 如果账号还没连店铺,会自动跳到 `/shops/new`。要看 Dashboard 有数据,请先连一个全托测试店,等首次同步完成(2–10 分钟,视 SKU 数量)或手动点 Dashboard「立即同步」。

**Q: 页面上看不到「活动」「调价」「发品」等功能?**

A: 本 MVP 明确砍掉了所有写入类功能,产品定位是**只读 BI 看板**。商家信任 Temu 官方卖家中心做写入操作,本应用只负责数据聚合和可视化。这些功能将在后续版本根据用户反馈决定是否补充。

**Q: 半托管店铺能用吗?**

A: 目前**只支持全托管**。Temu 半托管的销售 API(`bg.goods.salesv2.get` 对半托返回 `type not exists`)尚未向服务商开放。我们已在 Dashboard 明确提示,且店铺连接表单会在保存前做 shopType cross-check。

**Q: 数据多久更新一次?**

A: 默认每小时整点自动同步。用户可点「立即同步」触发即时拉取(202 Accepted,后台异步执行,前端 5 秒轮询检测到新数据自动刷新)。

**Q: Temu API 配额会被打爆吗?**

A: 单店每小时 1 次 salesv2 + 2 次 goods.list + 1 次 price.review,合计 4 次/店/小时。10 店 × 24h × 4 = 960 次/日,远低于常规配额。大规模(30+ 店)考虑分片 cron 或降频。
